# Projection Mapping Lab

Webカメラで検出した物理オブジェクトを、Matter.js の静的コリジョンとして使うリアルタイム・プロジェクションマッピング実験アプリです。

OpenCV.js で HSV 色検出を行い、検出したマーカーの輪郭を投影キャンバス座標へ変換して、ボールが跳ね返る物理デモとして表示します。

## セットアップ

```bash
npm install
```

## 起動

```bash
npm run dev -- --port 5173
```

ブラウザで開きます。

```
http://localhost:5173/
```

## 機能一覧

### 認識モード

プロジェクターとカメラの設置状況に合わせて、2つのキャリブレーションモードを切り替えられます。

#### 全体認識モード

カメラ映像にプロジェクターの投影範囲全体が映っている場合に使います。

```
カメラ映像上の4点 → プロジェクター Canvas 全体の4隅
```

カメラ映像内で指定した4点が、プロジェクター出力の全画面に対応します。

#### 部分認識モード

プロジェクターの投影範囲が広く、その一部だけがカメラに映っている場合に使います。

```
カメラ映像上の4点 → Interaction Area の4隅
```

カメラで見えている範囲を Canvas 全体に引き伸ばさず、Canvas 内に設定した **Interaction Area** という矩形領域に対応させます。Interaction Area の外側は余白として扱われ、その範囲ではリアルタイムのマーカー認識やコリジョン生成は行いません。

モードはコントロールパネルの「認識モード」から切り替えます。自動判定は行いません。

---

### Interaction Area（部分認識モード専用）

部分認識モードのとき、プロジェクター Canvas 内のどの領域をインタラクション対象にするかを設定します。

| 設定項目 | 説明 |
|---|---|
| X | Interaction Area の左端位置（px） |
| Y | Interaction Area の上端位置（px） |
| 幅 | Interaction Area の幅（px） |
| 高さ | Interaction Area の高さ（px） |

「エリア枠を表示」チェックをオンにすると、プレビューと投影出力の両方に Interaction Area をオレンジの破線枠で表示します。物理環境と合わせながら調整する際に使います。

デフォルト値は `x: 160, y: 90, width: 960, height: 540`（Canvas 中央に 75% の領域）です。

---

### キャリブレーション

「4点補正」ボタンを押してキャリブレーションモードに入り、カメラ映像内で4点を順番にクリックします。

```
1. 左上
2. 右上
3. 右下
4. 左下
```

4点が揃うとホモグラフィ行列が計算され、カメラ座標からプロジェクター座標への変換に使われます。

- **補正初期化**: キャリブレーション点をデフォルト（映像四隅）に戻します。
- **投影オフセット X / Y**: ホモグラフィ変換後の残ズレを数値で微調整します。

---

### 設定の保存・読み込み

#### キャリブレーション設定

「認識モード」セクションの保存・読み込みボタンで `calibration-settings.json` を出力・入力します。

保存される項目:

```json
{
  "mode": "full" | "partial",
  "cameraPoints": [...],
  "interactionArea": { "x": 160, "y": 90, "width": 960, "height": 540 },
  "offset": { "x": 0, "y": 0 },
  "showInteractionArea": true
}
```

プロジェクターとカメラの位置が固定されている場合は、前回の設定をそのまま読み込んで使えます。

#### 色検出設定

「色検出」セクションの保存・読み込みボタンで `color-settings.json` を出力・入力します。

---

### 色検出

OpenCV.js の HSV 色空間でマーカーを検出します。

| スライダー | 説明 |
|---|---|
| Hue | 検出する色相（OpenCV HSV: 0〜179） |
| 許容幅 | Hue の許容範囲 |
| 彩度 | 検出する彩度の下限（0〜255） |
| 明度 | 検出する明度の下限（0〜255） |

「スポイトで色を選択」ツールを使うと、カメラ映像内でクリックした点の色を自動で設定します。

「判定範囲を設定」ツールでカメラ映像内の矩形を指定すると、その範囲内だけで色検出を行います。

---

### プロジェクター出力

「プロジェクターへ出力」ボタンで別ウィンドウを開き、1280×720 の出力キャンバスを表示します。

- クリックでフルスクリーンになります。
- 出力ウィンドウにはボールとコリジョン枠だけが表示されます（カメラ映像は映りません）。
- 部分認識モードで「エリア枠を表示」がオンの場合、Interaction Area の枠も投影されます。

「左右反転」「上下反転」でプロジェクターの設置向きに合わせて映像を反転できます。

「テストパターン」でカラーバーとチェッカーボードを表示し、プロジェクターの色調整や位置合わせに使えます。

---

### カメラオーバーレイ（プレビューのみ）

コントロールパネルの「カメラオーバーレイ」スライダーで、プレビュー上のプロジェクター出力にカメラ映像を半透明で重ねて表示できます。

- 値: 0.00（非表示）〜 1.00（完全表示）、初期値 0.40
- プロジェクター出力ウィンドウには反映されません。

---

### 物理演算

Matter.js を使ったリアルタイム物理演算を行います。

- 検出されたマーカーの輪郭が静的コリジョンとして毎フレーム更新されます。
- ボールはコリジョンや壁に反発しながら落下します。
- 「ボール追加」ボタン、または「クリックでボール配置」ツールでボールを追加できます。
- 「クリックでコリジョン配置」ツールで手動の静的コリジョンを追加できます。
- 「コリジョン消去」で手動コリジョンをすべて削除します。
- ボールが Interaction Area（または Canvas）の外に出ると自動で削除されます。

---

### デモモード

カメラが使えない環境では自動でデモモードに切り替わります。ウォブルアニメーションする2つの仮想マーカーで動作確認できます。「デモ」ボタンで手動でも切り替えられます。
