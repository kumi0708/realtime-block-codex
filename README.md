# Projection Mapping Lab

Webカメラで検出した物理オブジェクトを Matter.js の静的コリジョンとして扱うリアルタイム・プロジェクションマッピング実験アプリです。

OpenCV.js で HSV 色検出を行い、マーカーの輪郭を投影キャンバス座標へ変換して、ボールが跳ね返る物理デモとして表示します。

---

## セットアップ・起動

```bash
npm install
npm run dev -- --port 5173
```

```
http://localhost:5173/
```

### Interaction Area（部分認識モード選択時のみ表示）

物理演算とマーカー認識が有効になる投影キャンバス上の領域です。  
台形など**矩形以外の任意の四角形**を設定できます。

| 項目 | 内容 |
|------|------|
| 4点設定 | 投影プレビューを4点順番にクリックして形状を確定する |
| 初期化 | デフォルト矩形（左上160,90 ／ 右下1120,630）に戻す |
| エリア枠を表示 | 投影プレビューと出力ウィンドウへのオレンジ枠描画を切り替える |

### 色検出

マーカーとして検出するHSV色の範囲を設定します。  
OpenCV.js の HSV 空間で「色相 ± 許容幅」「彩度 ≥ 下限」「明度 ≥ 下限」の3条件を満たすピクセルをマスクとして抽出します。

| スライダー | 範囲 | デフォルト | 意味 |
|------------|------|-----------|------|
| 検出FPS | 1〜60 | 60 | マーカー検出を実行するフレームレート。下げるとCPU負荷を減らせる |
| Hue（色相） | 0〜179 | 150 | 検出したいマーカーの色相値。OpenCV形式で0〜179（通常の0〜360を半分にした値）。赤=0/179付近、緑=60、青=120、マゼンタ=150 |
| 許容幅 | 4〜45 | 14 | 色相のブレ許容量。`Hue ± 許容幅` の範囲を検出対象とする。光の当たり方や素材のムラがある場合は大きめに設定する |
| 彩度（下限） | 0〜255 | 90 | この値以上の彩度のみ検出する。低くすると白・グレー・黒など色の薄いものも拾ってしまうため、誤検出が増える場合は上げる |
| 明度（下限） | 0〜255 | 80 | この値以上の明度のみ検出する。低くすると暗い色も拾う。暗い環境では下げると検出しやすくなる |

#### 調整のコツ

1. 「スポイトで色を選択」ボタンでカメラ映像のマーカー部分をクリックすると、その色から Hue・彩度・明度を自動設定できます。
2. 「検出マスク」プレビューを見ながら調整します。**白い部分が検出対象領域**です。マーカーだけが白くなる状態が理想です。
3. マーカー以外の部分も白くなる（誤検出）場合は、許容幅を狭めるか、彩度・明度の下限を上げます。
4. マーカーが検出されない場合は、許容幅を広げるか、彩度・明度の下限を下げます。
5. 設定が決まったら「保存」でJSONファイルに書き出しておくと次回読み込めます。

### 投影オフセット

ホモグラフィ変換後の残ズレを手動で微調整します。

| スライダー | 範囲 | デフォルト |
|------------|------|-----------|
| X | −160〜160 | 0 |
| Y | −120〜120 | 0 |

---

### カメラオーバーレイ

Projector Output プレビューにWebカメラ映像を半透明で重ねて表示します。  
4点補正で定義した領域の内側のみ描画されます。出力ウィンドウには表示されません。

| スライダー | 範囲 | デフォルト |
|------------|------|-----------|
| 不透明度 | 0.00〜1.00（0.05刻み） | 0.40 |

---

### ボール自動追加

指定した座標に一定間隔でボールを自動スポーンします。  
「自動スポーン」ボタンで開始し、「スポーン停止」で止めます。

| スライダー | 範囲 | デフォルト |
|------------|------|-----------|
| X（スポーン位置） | 0〜1280 | 640 |
| Y（スポーン位置） | 0〜720 | 40 |
| 間隔（秒） | 0.5〜30（0.5刻み） | 3.0 |
